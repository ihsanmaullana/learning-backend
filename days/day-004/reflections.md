# Day 4 — Reflections

## What went well

- Successfully ran my first `.js` file with Node.js.
- Successfully debugged my first error entirely on my own.

## What was hard

- Reading the error message for the first time was confusing. There is a lot of output
  (the full stack trace), and it took a moment to learn to look only at the first line
  that references my own file.

## One thing that clicked today

- Discovering that you can write and run JavaScript directly in the browser console.
  I always thought the console was only for displaying output — finding out it is a
  full interactive runtime was a genuine surprise.

## Questions I still have

- Why learn JavaScript before jumping straight to Node.js or TypeScript?
  Could I skip JavaScript and go directly to TypeScript from the start?

> *Note: The reason is that TypeScript compiles down to JavaScript — understanding the
> underlying language first makes TypeScript's type system and quirks far easier to reason
> about. Skipping JavaScript would mean debugging TypeScript errors without understanding
> what the compiled output actually does.*
