# Day 2 — Git Fundamentals

## Key Concepts

- **Git**: A version control system that tracks every change in your code
- **Repository (repo)**: A folder tracked by Git
- **Commit**: A "snapshot" of your code at a specific moment
- **Commit message**: A note explaining what changed and why
- **Remote**: A copy of your repo on a server (like GitHub)
- **Push**: Upload your commits to the remote

## Commands Learned

| Command | What it does |
|---------|-------------|
| `git init` | Initialize a new Git repository |
| `git status` | Check which files have changed |
| `git add` | Stage files for the next commit |
| `git commit` | Save a snapshot with a message |
| `git log` | View commit history |
| `git remote add` | Connect local repo to GitHub |
| `git push` | Upload commits to GitHub |
