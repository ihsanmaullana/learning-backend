# Day 1 — Terminal, File System & VS Code Setup

## Key Concepts

- **Terminal (CLI):** A text-based interface to communicate directly with the computer.
- **GUI vs CLI:** GUI uses visual elements (buttons, menus); CLI uses text commands. Most
  developers prefer the terminal because it gives direct system access and enables operations
  that are simply not possible through a GUI.
- **File System:** How files and folders are organized and navigated on a computer.

---

## Commands Learned

| Command   | What It Does                                    |
|-----------|-------------------------------------------------|
| `pwd`     | Print Working Directory — shows current location |
| `ls`      | List files and folders in the current directory  |
| `cd`      | Change Directory — navigate between folders      |
| `mkdir`   | Make a new directory (folder)                    |
| `touch`   | Create an empty file                             |
| `cat`     | Display the contents of a file                   |
| `clear`   | Clear the terminal screen                        |

---

## Reflections

Everything at this stage is relatively straightforward. The mental model of navigating
a file system through text commands took a little adjustment, but once it clicks it feels
more natural than clicking through a file explorer.
